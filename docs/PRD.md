# PPT 自动生成任务 PRD

## 1. 文档信息

- 项目名称：PPT 模板解析与自动生成系统
- 当前阶段：方案设计 / Python 首版实现规划
- 目标用户：需要根据需求文档快速生成项目汇报、方案介绍、架构说明类 PPT 的业务人员

## 2. 项目背景

当前已有一个模板文件 `templete.pptx`，其中包含多页固定版式页面。

期望通过 Python 实现一套自动化流程：

1. 解析模板 PPTX
2. 将每页规则抽取为 JSON
3. 结合需求文档与页面规则，按页调用大模型生成内容
4. 对流程图、架构图等图形内容进行文本定义后渲染为图片
5. 最终将全部页面结果汇总，生成新的 PPT 文件

## 3. 项目目标

### 3.1 核心目标

实现一个可复用的 PPT 生成流程，使系统能够：

- 自动解析模板中每页标题、文本框、表格和图片区域的结构、位置和样式规则
- 按页生成文本框内容、表格内容和 Mermaid 图定义
- 支持“有信息则生成、无信息则删除该页”的页面策略
- 支持按页并发调用大模型，线程池大小可配置
- 自动组装输出最终 PPT

### 3.2 成功标准

系统应满足以下结果：

- 可以稳定从 `templete.pptx` 的 `ppt/slides/*.xml` 提取出模板规则 JSON
- 模板规则 JSON 能清晰描述每页标题、文本框、表格和图片区域的位置信息与样式要求
- 单页大模型调用能够返回结构化结果，而不是自由文本
- 对于流程图、架构图、时序图等页面，能够生成 Mermaid 定义、渲染为 PNG，并插入 PPT
- 对于无内容页面，最终结果中可直接删除该页

## 4. 范围定义

### 4.1 本期范围

本期优先实现以下能力：

- 解析 `ppt/slides/*.xml` 为结构化 JSON
- 定义页面级规则和页面级生成协议
- 支持文本框、表格和 Mermaid 渲染图片三类内容
- 支持无内容页面直接删页
- 支持按页并发生成，线程池大小可配置
- 支持最终 PPT 组装

### 4.2 暂不纳入本期

以下能力暂不作为首版必须项：

- 备注栏写入
- 除 Mermaid 渲染图外的其他图片来源
- 复杂动画恢复
- 母版级动画或切换效果完整复制
- 图表数据驱动的原生 PowerPoint 图表编辑
- 原生形状、连接线、分组图形的自动编辑
- 人工拖拽式页面编辑器
- 多模板智能选择

## 5. 用户故事

### 5.1 作为业务人员

我希望上传一个需求文档，并基于已有模板自动生成 PPT，这样我就不需要手工逐页填写内容。

### 5.2 作为方案人员

我希望系统能够识别哪些页面适合当前需求，哪些页面不适合，并跳过不相关页面，这样最终 PPT 更精简。

### 5.3 作为后续维护人员

我希望图形页使用标准 Mermaid 语法生成，这样以后可以重新渲染或修改。

## 6. 总体流程

### 6.1 你提出的原始流程

1. 解析 `templete.pptx`
2. 将模板规则转为 JSON
3. 读取需求文档
4. 按页调用大模型生成内容
5. 对流程图/架构图先输出图定义，再转图片插入 PPT
6. 收集全部页面结果
7. 组装生成最终 PPT

### 6.2 已确认的首版流程

当前已确认的首版流程如下：

1. 仅解析 `ppt/slides/*.xml`，输出 `template_rules.json`
2. 读取纯文本需求文档全文，篇幅控制在 1 万字以内
3. 将“需求全文 + 单页规则”发送给大模型，按页独立生成
4. 模型先判断该页是否有足够信息；若无内容则返回跳过
5. 若页面为图形页，模型输出 Mermaid 定义
6. 渲染脚本使用 Mermaid CLI 将 Mermaid 转为 PNG 图片
7. 使用 Python 脚本把文本框、表格和 Mermaid 图片回填到 PPT
8. 对 `should_generate = false` 的页面直接删页
9. 输出最终 PPT

### 6.3 流程说明

- V1 不做摘要、不做切片，优先使用需求全文，避免信息失真
- 图形页首版统一使用 Mermaid，范围限定为架构图、流程图、时序图
- 除文本框、表格和 Mermaid 图片外，其他原生 PowerPoint 图元暂不进入自动生成范围

## 7. 模板解析需求

## 7.1 输入

- 模板文件：`templete.pptx`

## 7.2 解析范围

按当前业务约束，V1 只从 `ppt/slides/*.xml` 抽取模板规则，其他 OOXML 部件不作为规则来源。

当前首版重点解析以下信息：

- 页面标题
- 文本框
- 表格
- 图片区域（用于插入 Mermaid 渲染图）
- 每个元素的位置、尺寸和基础样式

如果后续发现个别样式未在 slide XML 中显式给出，再单独扩展解析范围。

## 7.3 页面规则 JSON 必须包含的信息

每一页至少包含：

- 页面编号
- 页面名称
- 页面标题文本
- 页面标题位置与尺寸
- 页面标题样式
- 页面用途判断
- 页面元素清单
- 页面是否支持 Mermaid 图生成

## 7.4 页面元素必须抽取的信息

每个元素至少包含：

- 元素唯一 ID
- 元素类型
- 所在页码
- 位置 `x/y`
- 尺寸 `width/height`
- 层级顺序
- 是否可编辑
- 默认内容
- 样式信息
- 内容要求

其中元素类型至少覆盖：

- `title`
- `text`
- `table`
- `image`

## 7.5 样式信息建议包含

- 字体
- 字号
- 字重
- 颜色
- 对齐方式
- 行距
- 段前段后
- 缩进
- 内边距
- 背景色
- 边框色
- 边框粗细
- 圆角或几何类型
- 自动适配方式
- 主题色引用

## 7.6 内容要求建议字段

为了让大模型可直接使用，页面元素规则中建议加入以下推导字段：

- `role`：元素角色，例如标题、正文、表格、图片区
- `content_requirement`：对该元素内容的文字要求
- `max_chars`：适用于文本框
- `table_schema`：适用于表格
- `max_rows` / `max_cols`：适用于表格容量约束
- `diagram_kind`：适用于 Mermaid 图，例如 `architecture`、`flowchart`、`sequence`
- `fill_strategy`：`replace_text`、`fill_table`、`render_mermaid_to_image`

## 8. 建议的模板规则 JSON 结构

```json
{
  "template": {
    "file_name": "templete.pptx",
    "slide_size": { "cx": 12192000, "cy": 6858000 },
    "slide_count": 48,
    "theme": {
      "fonts": {},
      "colors": {}
    }
  },
  "pages": [
    {
      "page_no": 1,
      "page_name": "封面页",
      "page_purpose": "封面",
      "title": {
        "text": "默认标题",
        "bbox": { "x": 0, "y": 0, "w": 0, "h": 0 },
        "style": {}
      },
      "elements": [
        {
          "id": "shape_1",
          "type": "text",
          "role": "subtitle",
          "bbox": { "x": 0, "y": 0, "w": 0, "h": 0 },
          "default_text": "默认文案",
          "style": {},
          "content_requirement": "填写项目副标题",
          "fill_strategy": "replace_text"
        }
      ]
    }
  ]
}
```

## 9. 大模型分页生成需求

## 9.1 调用原则

- 每一页单独调用一次大模型
- 输入应包含“需求文档内容 + 单页模板规则”
- 模型应先判断该页是否适合当前需求
- 若不适合，应明确返回跳过原因

## 9.2 单页输入

单页大模型输入建议包含：

- 页面基础信息
- 页面用途
- 标题要求
- 元素清单
- 每个元素的内容要求
- 需求文档全文
- 输出格式要求

## 9.3 单页输出

单页输出必须是结构化 JSON，而不是纯自然语言。

建议格式：

```json
{
  "page_no": 15,
  "should_generate": true,
  "skip_reason": "",
  "elements": [
    {
      "id": "title_1",
      "type": "text",
      "content": "总体技术架构"
    },
    {
      "id": "image_1",
      "type": "image",
      "image_source_type": "mermaid",
      "diagram_kind": "architecture",
      "mermaid_syntax": "architecture-beta",
      "mermaid_source": "architecture-beta\ngroup api(cloud)[API]\nservice gateway(internet)[Gateway]\n..."
    }
  ]
}
```

## 9.4 页面跳过规则

大模型需要支持：

- 需求文档无充分信息时跳过
- 页面用途与当前需求不匹配时跳过
- 图形页缺乏关键节点或关系时跳过
- 表格页缺乏足够结构化信息时跳过

## 10. 图形页生成方案

## 10.1 目标

流程图、架构图等复杂图形内容，不直接由脚本逐个创建原生形状，而是由大模型先输出图定义，再渲染为图片插入 PPT。

## 10.2 推荐方案

首版建议统一使用 **Mermaid** 作为图定义语法。

原因：

- Mermaid 官方已支持 `architecture-beta`、`flowchart`、`sequenceDiagram`
- 语法简单
- 学习成本低
- 适合大模型稳定输出
- Mermaid CLI 可直接渲染 PNG 或 SVG
- 与 PPT 插图流程匹配

## 10.3 图形页流程

1. 页面规则中识别图片区域
2. 大模型输出 Mermaid 定义
3. 脚本将 Mermaid 文本写入 `.mmd` 文件
4. 使用 Mermaid CLI 渲染，例如：`mmdc -i input.mmd -o output.png -t neutral -b transparent`
5. 使用 `python-pptx` 的 `slide.shapes.add_picture(image_file, left, top, width, height)` 将图片插入模板指定区域
6. 保留 `.mmd` 与页面 JSON 作为中间产物

## 10.4 首版备注策略

- V1 不写备注栏
- 若后续需要追溯 Mermaid 原文，以中间 `page_xx.json` 和 `.mmd` 文件为准

## 11. PPT 组装需求

## 11.1 输入

- 原始模板 PPTX
- 模板规则 JSON
- 所有页面生成 JSON
- 渲染好的图片文件

## 11.2 输出

- 最终 PPTX 文件
- 可选日志文件
- 可选页面结果汇总 JSON

## 11.3 组装规则

- 按模板页顺序处理
- 对 `should_generate = false` 的页面直接删页
- 仅处理文本框、表格和 Mermaid 渲染图片
- 文本元素按规则回填
- 表格元素按行列结构回填
- 图片元素仅处理 Mermaid 渲染结果
- 保持模板页原有版式、边距和整体视觉风格

## 12. 非功能需求

### 12.1 可维护性

- 规则 JSON 结构清晰
- 单页生成结果格式固定
- 出错时可定位到具体页和具体元素

### 12.2 可扩展性

- 后续可以支持多个模板
- 后续可以支持不同图定义语法
- 后续可以支持多模型切换

### 12.3 稳定性

- 单页失败不应影响全部流程
- 图渲染失败应可降级
- 生成结果应保留中间文件，便于排查

### 12.4 成本控制

- 支持按页并发
- 支持通过 `max_workers` 配置线程池大小
- 支持跳过不必要页面
- 需求全文长度控制在 1 万字以内

## 13. 首版建议的技术模块

### 13.1 模板解析模块

职责：解析 `templete.pptx` 并输出 `template_rules.json`

### 13.2 需求读取模块

职责：读取纯文本需求全文并做基础标准化

### 13.3 页面生成模块

职责：按页并发调用大模型并输出 `page_xx.json`

### 13.4 图渲染模块

职责：把 Mermaid 文本转为图片

### 13.5 PPT 回填模块

职责：将页面生成结果回写到最终 PPTX

### 13.6 并发调度模块

职责：根据 `max_workers` 管理页面级并发调用、重试和结果汇总

## 14. 验收标准

首版交付应至少满足：

- 能从 `ppt/slides/*.xml` 提取全部页面规则
- 规则 JSON 中能看到每页标题、文本框、表格和图片区域位置信息
- 至少支持文本框、表格、Mermaid 图片三类内容
- 支持单页生成与单页删页
- 能将 Mermaid 架构图、流程图、时序图渲染后插入 PPT
- 能输出一个最终 PPT 文件

## 15. 当前已确认的实现约束

- 模板规则只从 `ppt/slides/*.xml` 读取
- 需求输入为纯文本全文，长度不超过 1 万字，V1 不做摘要或切片
- Mermaid 作为唯一图定义语法，首版支持架构图、流程图、时序图
- V1 只处理文本框、表格、图片文件三类元素
- 某页无内容时直接删页
- 表格超长优先通过提示词限制，不优先做后处理
- PPT 中的图片仅来自 Mermaid 渲染结果
- V1 不写备注栏
- 需要并发调用模型，线程池大小可配置

## 16. 当前推荐的实现参数

- Mermaid 渲染默认输出 PNG，背景透明
- 页面级并发通过 `max_workers` 控制
- 中间产物至少保留 `page_xx.json`、`page_xx.mmd`、`diagram_xx.png`
- 模型输出必须严格使用 JSON，避免自由文本

## 17. 推荐的下一步

建议按下面顺序推进：

1. 先实现模板规则抽取脚本
2. 再定义单页生成 JSON 协议和提示词约束
3. 然后实现 Mermaid 渲染与图片插入
4. 再实现并发调度与 PPT 组装
5. 最后补充删页逻辑与异常处理

## 18. 本次结论

首版边界已经明确，可以直接进入编码阶段。

当前已经确定的关键实现原则是：

- 只解析 `ppt/slides/*.xml`
- 需求全文直接输入模型
- 图形页统一使用 Mermaid
- 首版只处理文本框、表格和 Mermaid 图片
- 无内容页面直接删页
- 并发调用模型且线程池大小可配置

在这个边界下，后续 Python 首版实现会更直接，也更容易验证结果。
