

# **ppt-master 试用报告**

本次试用围绕 `templete.pptx` 与 `req.txt` 两类输入展开，重点验证 `ppt-master` 在“模板复用 + 长需求文档生成”场景下的成稿能力、稳定性、资源消耗以及可交付性。整体来看，工具能够完成从模板解析、页选规划、内容填充到导出校验的完整闭环，适合用于内部汇报、需求说明、技术方案等标准化材料；但其代价是较高的生成成本、较长的处理时间，以及模板一致性与可编辑性之间的明显冲突。

## **一、试用结论**
- `ppt-master` 能较好地完成结构化 PPT 的自动生成，尤其适用于有固定模板、内容边界清晰的正式汇报材料。
- 工具更偏向“模板优先、成稿优先”，不是“可编辑形状优先”的传统 PPT 设计器。
- 对于需要强一致视觉、强编辑能力、强人工微调的场景，当前版本不算理想。

## **二、关键指标**
| 项目 | 试用结果 |
| --- | ---: |
| 成稿规模 | 约 20 页 PPT |
| 输入消耗 | 约 3000w token |
| 输出消耗 | 约 20w token |
| 工具调用次数 | 约 100 次 |
| 总耗时 | 约 40-50 分钟 |
| 中间产物保留 | 是 |
| 图片替换 | 需手动指定 |
| SVG 风格继承 | 不会自动参考前图风格 |
| 可编辑形状 | 与模板复用存在冲突 |

## **三、主要观察**
- 使用 PPT 模板后，系统更倾向于保持模板版式；如果强行要求可编辑形状，往往会重新生成一个不再贴合模板的版本。
- 模板中的图片不会自动替换，需要人工提供或手动指定映射关系。
- 生成的 SVG 转图片后，默认不会继承前一张图的视觉风格，图表/插图一致性需要额外约束。
- 长文档生成过程呈明显的多阶段链路，适合“先结构、后内容”的工作方式。
- 中文长文本、表格和编码环境对结果影响较大，前置整理很重要。

## **四、主要问题与风险**
- **模板一致性 vs 可编辑性冲突**：两者很难同时满足；一旦强调可编辑形状，就会不使用给到的模版而去自己设计。
- **视觉连续性不足**：图片和 SVG 风格缺少自动继承机制，容易出现同一套 PPT 内风格不统一。
- **人工介入仍然必要**：图片替换、关键页校对、局部版式微调仍需要人工参与。
- **资源成本较高**：输入/输出 token、工具调用次数和时间成本都不低，不适合高频批量任务。
- **适用面有限**：对标准化汇报材料友好，对强创意、强交互、强编辑诉求场景不够友好。

## **五、中间产物保留情况**
本次试用保留的中间产物包括：

- `sources/`：原始输入文件副本，如 `req.txt`、`templete.pptx`
- `analysis/`：模板解析结果与页级分析结果，如 `slide_library.json`
- `analysis/page_layout_rationale.md` 或同类页布局说明文件
- `analysis/fill_plan.json`：填充计划
- `validation/check_report.json`：容量/一致性检查结果
- `validation/readback.md`：导出后的回读校验文件
- `exports/`：最终导出的 PPTX 成品
- 如有需要，还可保留模板提取后的 Markdown 文件，如 `templete.md`


## **六、执行过程中的主要错误/报错**
试用过程中出现的主要问题如下：

| 问题类型 | 现象 | 影响 | 处理建议 |
|---|---|---|---|
| 控制台编码问题 | 出现 `UnicodeEncodeError`，中文输出在 PowerShell 环境下不稳定 | 影响中文内容直接打印与调试 | 统一使用 UTF-8 输出，必要时写入文件后再读取 |
| 命令引用/换行问题 | `python -c`、PowerShell here-string 组合时出现语法或引用错误 | 影响脚本临时执行 | 改为脚本文件执行或严格统一引号规则 |
| 输出截断/乱码 | 大段 JSON 或长文本直接打印时出现截断、乱码或不完整输出 | 影响模板解析和调试 | 改为落盘到文件，再按文件读取 |
| 模板路径歧义 | PPTX 文件与模板目录路径概念混淆，流程一度在模板流与自由设计之间反复确认 | 增加前期确认成本 | 统一输入规范，明确“模板目录”和“PPTX 文件”边界 |
| 表格编辑警告 | 若表格页未补齐 `table_edits`，会出现“原始模板内容保留”类警告 | 影响成稿完整性 | 所有使用表格的页都应补齐表格映射 |
| 长链路重复确认 | 目录、页选、图形策略多次确认 | 增加总耗时 | 建议先固化结构，再批量生成 |

说明：
- 这些问题多数属于“流程型问题”或“环境型问题”，没有造成最终流程完全中断，但显著增加了调试和人工确认成本。


## **七、总体评价**
- `ppt-master` 的优势在于能把长文档快速组织成结构化 PPT，适合标准化交付。
- 它的短板主要集中在可编辑性、图片一致性和生成成本上。
- 综合判断：**建议将其作为公司内部标准化 PPT 的初稿生成工具使用，不建议作为高可编辑、高定制交付的唯一生成手段。**
